4// ============================================================================
5// Width-2 (PIC / CIC) scatter / gather auto-tuning pre-pass.
6//
7// Default behaviour: NO sweep is run. Sensible per-exec-space defaults are
8// seeded into TileSizeCache and GatherCache by Ippl::initialize, so the
9// first scatter/gather call uses a known-good configuration without paying
10// any benchmarking cost or touching the filesystem.
11//
12// Opt in to the sweep through the IPPL_AUTO_TUNE env var:
13//
14// IPPL_AUTO_TUNE=1 (or "quick") - small candidate set on a single
15// 32^3 grid; finishes in ~seconds.
16// IPPL_AUTO_TUNE=full (or "2") - much broader sweep across grid sizes
17// {32, 64, 128}, particle densities
18// {0.5, 2, 8, 32} ppc, larger tile /
19// team / oversubscription / z_batch
20// candidate sets, longer per-config
21// measurement. Each density bucket gets
22// its own CSV row so the runtime
23// density-aware lookup picks the closest
24// match. Tens of seconds to a few
25// minutes on a GPU.
26//
27// Anything else (or unset) is treated as no-op. Progress is reported
28// through ippl::Info at info level >= 1 (`--info 1`).
29//
30// On Serial we skip the benchmark entirely (there is nothing to tune -
31// Atomic with team_size 1 is the only valid config) and just write a
32// single Atomic row so subsequent loads are consistent.
33// ============================================================================
